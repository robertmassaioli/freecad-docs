## Local development

### Prerequisites

- Python 3.10 or later
- `pip`
- Node.js (optional — only needed if you want to use `npx http-server`)

### First-time setup

Create and activate a virtual environment, then install the dependencies:

```bash
python3 -m venv .venv
source .venv/bin/activate      # macOS / Linux
# .venv\Scripts\activate       # Windows

pip install -r requirements.txt
```

### Option A — Live-reload dev server (recommended while writing)

```bash
.venv/bin/mkdocs serve
```

Open **http://127.0.0.1:8000/** in your browser. The page reloads automatically
whenever you save a Markdown file. Press `Ctrl+C` to stop.

### Option B — Build once and browse the static output

Build the static site into `site/`:

```bash
.venv/bin/mkdocs build
```

Then serve it with whichever tool you prefer:

```bash
# Using npx (Node.js required)
npx http-server site/ --port 8000 --open

# Using Python's built-in server (no extra deps)
python3 -m http.server 8000 --directory site/
```

Open **http://localhost:8000/** in your browser.

> **Note:** `mkdocs serve` (Option A) is better during active writing because
> it handles the `site_url`-relative asset paths correctly and hot-reloads.
> Option B is useful for a final check of the exact static output that will
> be deployed.

## Contributing

1. Read `agents.md` for the full authoring guide — it covers source
   references, OSS-only Material features, writing conventions, and how
   to look things up in the FreeCAD source.
2. Read `AI-planning/tool-documentation-format.md` before writing or
   reviewing any tool page.
3. Run `mkdocs build` and confirm **0 errors** before opening a PR.

---

## Deployment

Every push to `main` automatically builds and deploys to GitHub Pages via
`.github/workflows/deploy.yml`. No manual steps needed.

To trigger a deploy manually: **Actions → Deploy to GitHub Pages → Run workflow**.
